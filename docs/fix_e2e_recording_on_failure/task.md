# タスク: E2Eテスト失敗時の録画確認対応

## 概要
ユーザーは Playwright による E2E テスト (`scrum.spec.ts`) が失敗した際にも、操作画面の録画（ビデオ）を確認できるようにすることを求めています。現在はテストが正常に完了した最後にのみ動画添付処理が行われているため、途中で失敗すると動画がレポートに含まれません。

## 現状の課題
- `scrum.spec.ts` では、テストケースの末尾で動画のパスを取得し `test.info().attach()` を行っている。
- 途中でアサーションエラーやタイムアウトが発生すると、末尾のコードに到達せず、動画が添付されない。
- また、動画の保存（`context.close()`）が `afterAll` で行われているが、失敗時のフローと連携していない可能性がある。

## 対応方針
1. 動画の添付処理を `test.afterEach` フックに移動する。
   - `afterEach` はテストの成功・失敗に関わらず実行される。
2. `test.afterAll` で行っていたコンテキストのクローズ処理も `test.afterEach` に移動し、動画の保存を確実に行う。
3. コンテキストクローズ後に動画パスを取得し、レポートに添付する実装にする。
