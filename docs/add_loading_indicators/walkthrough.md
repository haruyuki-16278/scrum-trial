# 検証手順: ローディング・スケルトン表示の追加

## 変更内容の確認
以下の手順で変更が正しく機能しているかを確認する。

### 1. タスク追加のローディング
1.  プロダクトバックログの入力欄にテキストを入力する。
2.  「+」ボタンをクリックする。
3.  **期待値**: 追加処理が完了してリストに反映されるまでの間、ボタンがスピナー（またはLoading文字）に変わり、連打できないようになること。

### 2. タスク移動のローディング
1.  プロダクトバックログのタスクの「→ Sprint」ボタンをクリックする。
2.  **期待値**: ボタンがローディング状態になり、スプリントバックログに移動するまで操作不能になること。
3.  スプリントバックログのタスクの「← PBL」ボタンをクリックする。
4.  **期待値**: ボタンがローディング状態になり、プロダクトバックログに戻るまで操作不能になること。

### 3. ロール切り替えのローディング
1.  チームパネルで自分のロールアイコンをクリックする。
2.  **期待値**: アイコンの中央にスピナーが表示されるか、アイコン全体がDimして、ロールが切り替わるまで待機状態に見えること。

### 4. 異常系（任意）
*   意図的にネットワーク遅延を発生させた場合でも、ローディング表示が継続し、完了後に元に戻ることを確認する（目視確認のみでも可）。
