# Scrum Trial App

このリポジトリは、以下の勉強会資料（スライド #30 ~ #42）で紹介されている「スクラム開発体験ゲーム」をオンラインでスムーズに実施するためのWebアプリケーションです。

- **資料URL**: [https://jigintern.github.io/study_session_materials/scrum-trial-2025/slide.html](https://jigintern.github.io/study_session_materials/scrum-trial-2025/slide.html)
- **対象範囲**: ハッシュ #30 ~ #42 (スクラムシミュレーションゲーム)

## アプリケーションの機能

現状のアプリケーションは、ゲームを進行するための以下の基本的な機能を提供しています：

- **リアルタイム同期**: チームメンバー間での状態共有
- **プロダクトバックログ (PBL) 管理**: タスクの追加、並び替え、スプリントへの移動
- **スプリントバックログ (SBL) 管理**: 進行状況の可視化、完了管理
- **ダイスロール機能**:
  - タスクを指定して振る: 出目の分だけ進捗が進みます（完了目標値: 12）。
  - フリーロール: イベントやボーナス判定用に使用できます。
- **共有メモ (KPT)**: ※現バージョンでは閲覧のみ

## 実行方法

```bash
deno task dev
```

## ゲームのルール概要

1.  **役割**: PO、SM、開発チーム
2.  **スプリント**: 全3回（各スプリントはゲーム内時間で4日間）
3.  **進捗**: 1日1回ダイスを振り、出目を進捗として加算。合計12以上でタスク完了。
4.  **イベント**:
    - **プランニング**: タスクの作成とアサイン
    - **デイリースクラム**: ダイスロール（作業）
    - **レビュー**: 成果確認
    - **レトロスペクティブ**: KPT実施とボーナス判定（SMがサイコロを振り偶数なら次スプリントでボーナス）
