# AIエージェント向け行動指針 (AGENTS.md)

このプロジェクトでは、コードの可読性と親しみやすさを向上させるため、**ソースコード内の命名に日本語を使用すること**を強く推奨する特別ルールを採用しています。
AIエージェントは、コードの生成・修正を行う際、以下のルールを厳守してください。

## 命名規則 (日本語化ルール)

### 1. 基本方針
すべての識別子（変数名、関数名、クラス名、型名、メソッド名など）は、原則として**日本語（漢字・ひらがな・カタカナ）**で命名してください。

### 2. 具体的な命名例

| 種類 | 英語の例 (非推奨) | 日本語の例 (推奨) |
| :--- | :--- | :--- |
| **変数** | `const count = 0;` | `const カウント = 0;`<br>`const ユーザー数 = 0;` |
| **関数** | `function getUser()` | `function ユーザー取得()` |
| **引数** | `(userId: string)` | `(ユーザーID: string)` |
| **型/IF** | `interface User` | `interface ユーザー`<br>`type ユーザー情報 = ...` |
| **コンポーネント** | `function UserProfile()` | `function ユーザープロフィール画面()` |
| **フック** | `useSession()` | `useセッション()` |

### 3. 例外（英語のままにすべき箇所）
互換性や技術的な制約により、以下のケースは英語の使用を許可（または強制）します。

1.  **JSONデータのキー / 通信プロトコル**
    *   APIで送受信するJSONのプロパティ名（バックエンドとフロントエンドの契約）。
    *   例: `{ type: "JOIN", payload: { ... } }` の `type` や `payload`。
2.  **外部ライブラリ・フレームワークの仕様**
    *   Reactのライフサイクルメソッド (`useEffect`, `useState` など) や、フレームワークが規定しているファイル名・関数名。
    *   HTML属性 (`className`, `style`, `onClick` など)。
3.  **ファイル名**
    *   ファイル名は基本的に英語（小文字ケバブケースやパスカルケース）を維持するが、ユーザーの指示があれば日本語ファイル名も許容する。基本は `main.ts`, `App.tsx` のように英語とする。

### 4. コメント
コメントは全て**日本語**で記述してください。

---

## 開発プロセス

コードを修正する際は、既存の英語の変数が残っている場合、積極的に日本語へリファクタリングすることを推奨します（ただし、変更による影響範囲が大きい場合はユーザーに確認すること）。

## ドキュメント管理ルール

### 作業ログの保存
*   タスクの作業ログ（`task.md`, `implementation_plan.md` 等）を保存するフォルダを作成する場合、フォルダ名の先頭には必ず `_`（アンダースコア）を付けてください。
    *   これは `.gitignore` によってGit管理対象外とするためです。
    *   例: `docs/_feature_implementation`
*   逆に、成果物となる恒久的なドキュメント（仕様書など）を置くフォルダには `_` を付けないでください。
